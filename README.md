# DatoCMS plugins repository

This repository provides samples of DatoCMS plugins developed using the [Plugins SDK](https://www.datocms.com/docs/building-plugins/sdk-reference).

### List of plugins

- [AI Translations](https://github.com/datocms/plugins/tree/master/ai-translations/): Translate fields and records directly on DatoCMS using AI
- [Alt Text AI](https://github.com/datocms/plugins/tree/master/alt-text-ai/): Generate alt texts for your assets in a single click
- [Asset Optimization](https://github.com/datocms/plugins/tree/master/asset-optimization/): Mass apply optimizations to your DatoCMS assets
- [Automatic Environment Backups](https://github.com/datocms/plugins/tree/master/automatic-environment-backups/): Creates daily and weekly backups of your main environment automatically
- [Block to Links](https://github.com/datocms/plugins/tree/master/block-to-links/): Convert blocks to links
- [Bulk Change Author](https://github.com/datocms/plugins/tree/master/bulk-change-author/): Change the author of multiple records at a time on tabular view
- [Character Counter](https://github.com/datocms/plugins/tree/master/character-counter/): Show a character counter on fields with validations on length
- [Conditional Fields](https://github.com/datocms/plugins/tree/master/conditional-fields/): Show/hide fields when you toggle a checkbox boolean field
- [Content Calendar](https://github.com/datocms/plugins/tree/master/content-calendar/): Explore your editorial calendar, right inside DatoCMS
- [DALL-E](https://github.com/datocms/plugins/tree/master/dall-e/): Generate images with configured providers and insert them directly inside your project
- [Delete Assets Option](https://github.com/datocms/plugins/tree/master/delete-assets-option/): Prompts users to delete assets only referenced in the records they are deleting
- [Delete Unused Assets](https://github.com/datocms/plugins/tree/master/delete-unused-assets/): Delete all unused assets with the click of a button
- [Disabled Field](https://github.com/datocms/plugins/tree/master/disabled-field/): Disable a field that you don't want anyone to edit, works with every field type
- [Field Anchor](https://github.com/datocms/plugins/tree/master/field-anchor-menu/): Creates a sidebar anchor menu that links to your fields
- [Import/Export Schema](https://github.com/datocms/plugins/tree/master/import-export-schema/): Export models/block models as JSON for reimport into another DatoCMS project
- [Inverse Relationships](https://github.com/datocms/plugins/tree/master/inverse-relationships/): Displays inverse relationships on a record's page
- [Locale Duplicate](https://github.com/datocms/plugins/tree/master/locale-duplicate/): Duplicate the content from one locale to another
- [Lorem Ipsum Generator](https://github.com/datocms/plugins/tree/master/lorem-ipsum/): Automatically fill your textual fields with dummy content
- [Media Layouts](https://github.com/datocms/plugins/tree/master/media-layouts/): Select an asset together with a layout configuration
- [Sidebar Notes](https://github.com/datocms/plugins/tree/master/notes/): Add Post-it notes to your records sidebar
- [Project Exporter](https://github.com/datocms/plugins/tree/master/project-exporter/): Export records and assets from your project directly from your browser
- [Project-wide Stage Viewer](https://github.com/datocms/plugins/tree/master/project-wide-stage-viewer/): View all records across multiple models belonging to a workflow stage
- [Record Auto-save](https://github.com/datocms/plugins/tree/master/record-auto-save/): Auto-save your records after a time interval
- [Record Bin](https://github.com/datocms/plugins/tree/master/record-bin/): Manage and create a backup of deleted records
- [Record Comments](https://github.com/datocms/plugins/tree/master/record-comments/): Leave comments under a record with your project collaborators
- [Rich Text TinyMCE](https://github.com/datocms/plugins/tree/master/rich-text-tinymce/): Custom TinyMCE configuration for multi-paragraph fields
- [Schema ERD](https://github.com/datocms/plugins/tree/master/schema-erd/): Generate Graphviz ERD diagrams of your project schema in SVG, PDF and DOT formats
- [SEO Analysis](https://github.com/datocms/plugins/tree/master/seo-readability-analysis/): Runs SEO/Readability analysis using YoastSEO.js on your frontend everytime you make a change to a record
- [Shopify Product](https://github.com/datocms/plugins/tree/master/shopify-product/): Search and select Shopify products
- [Slug Redirects](https://github.com/datocms/plugins/tree/master/slug-redirects/): Registers all slug changes conveniently to manage redirects
- [Star Rating Editor](https://github.com/datocms/plugins/tree/master/star-rating-editor/): Presents integer fields as star rating widgets
- [Table Editor](https://github.com/datocms/plugins/tree/master/table-editor/): Transforms any JSON field into a table editor
- [Tag Editor](https://github.com/datocms/plugins/tree/master/tag-editor/): Transforms any string and JSON field into a tag editor
- [Sidebar Todos](https://github.com/datocms/plugins/tree/master/todo-list/): Add To-Do lists to your records sidebar
- [Tree-like Slugs](https://github.com/datocms/plugins/tree/master/tree-like-slugs/): Pass down slugs to child records in models with Hierarchical Sorting
- [Unsplash](https://github.com/datocms/plugins/tree/master/unsplash/): Search free high-resolution photos on Unsplash and insert them into your project
- [Web Previews](https://github.com/datocms/plugins/tree/master/web-previews/): Shows links to the matching webpage previews on selected records
- [Yandex Translate](https://github.com/datocms/plugins/tree/master/yandex-translate/): Automatically translate your content into secondary languages

# What is DatoCMS?

<a href="https://www.datocms.com/"><img src="https://www.datocms.com/images/full_logo.svg" height="60" alt="DatoCMS - The Headless CMS for the Modern Web"></a>

[DatoCMS](https://www.datocms.com/) is the REST & GraphQL Headless CMS for the modern web.

Trusted by over 25,000 enterprise businesses, agencies, and individuals across the world, DatoCMS users create online content at scale from a central hub and distribute it via API. We ❤️ our [developers](https://www.datocms.com/team/best-cms-for-developers), [content editors](https://www.datocms.com/team/content-creators) and [marketers](https://www.datocms.com/team/cms-digital-marketing)!

**Why DatoCMS?**

- **API-First Architecture**: Built for both REST and GraphQL, enabling flexible content delivery
- **Just Enough Features**: We believe in keeping things simple, and giving you [the right feature-set tools](https://www.datocms.com/features) to get the job done
- **Developer Experience**: First-class TypeScript support with powerful developer tools

**Getting Started:**

- ⚡️ [Create Free Account](https://dashboard.datocms.com/signup) - Get started with DatoCMS in minutes
- 🔖 [Documentation](https://www.datocms.com/docs) - Comprehensive guides and API references
- ⚙️ [Community Support](https://community.datocms.com/) - Get help from our team and community
- 🆕 [Changelog](https://www.datocms.com/product-updates) - Latest features and improvements

**Official Libraries:**

- [**Content Delivery Client**](https://github.com/datocms/cda-client) - TypeScript GraphQL client for content fetching
- [**REST API Clients**](https://github.com/datocms/js-rest-api-clients) - Node.js/Browser clients for content management
- [**CLI Tools**](https://github.com/datocms/cli) - Command-line utilities for schema migrations (includes [Contentful](https://github.com/datocms/cli/tree/main/packages/cli-plugin-contentful) and [WordPress](https://github.com/datocms/cli/tree/main/packages/cli-plugin-wordpress) importers)
