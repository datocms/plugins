# Changelog

- 3.7.1: Reliability and field-exclusion fixes on top of the 3.7.0 quality-control work, found in an adversarial review and validated end-to-end. **Field exclusion now works for every vendor.** The "Fields to be excluded from translation" list in plugin settings was only populated for OpenAI (and only once an OpenAI key was entered), leaving it empty for DeepL, Gemini, and Anthropic users — so nothing could be excluded. It now loads for every vendor, and lists fields **inside blocks** (shown as "_field_ (_Block_ block)"), so a single nested field can be left untranslated; the translation engine already honored those exclusions in both the single-record and bulk flows. **A single field failure no longer sinks a whole record when translating into a new locale.** Leaving a failed field out of the write produced an inconsistent locale set that DatoCMS rejected with `INVALID_LOCALES`, losing the record's successfully-translated fields too; now the record partial-saves (the failed field is left empty and flagged) while an existing translation is still never overwritten with an empty value. **An invalid or expired provider API key now pauses the bulk run** (surfacing the reason so you can fix the key), instead of silently failing every field — an auth error was being misclassified and its pause safeguard disabled. Other fixes: chat-model responses where an array element comes back untranslated (kept as its source text) are now flagged for review instead of shipping silently; a wrong-segment-count response is treated as a reviewable warning rather than a hard failure (so correctly-saved multi-part fields are no longer miscounted as failed); `Retry-After: 0` (or a past retry hint) now falls back to exponential backoff instead of hammering the endpoint; cancelling a bulk run mid-field now stops the in-flight record's remaining work and preserves the partial report; the length-limit pre-check now also catches values under a `min` (or not exactly `eq`) length, not just over `max`; a transient failure fetching an upload's default alt/title metadata is retried instead of disabling enrichment for the rest of the session; asset alt/title fallback now reads the upload's default metadata in **both** the legacy locale-first and the new field-first ("non-localized focal points") shapes, so alt/title enrichment keeps working on projects created after 2026-06-11 (previously it silently stopped); an over-long SEO title/description that gets trimmed is now flagged; the end-of-run report no longer double-counts a quality-control flag or mislabels a failed record's reason as a mere warning; the CSV export filename uses your local date; record labels with emoji are truncated safely; and the report panel now follows Dark Mode correctly. A second adversarial review tightened this batch further: the "came back untranslated" review flag no longer double-reports the same field, no longer fires on a legitimately-empty segment, and — on long fields translated in chunks — is reported once against the field's real length instead of once per chunk with a chunk-relative count; over-long SEO titles/descriptions and long record labels are now trimmed on a grapheme boundary, so an emoji or flag is never split (which for the SEO field would have persisted a broken character); and the "Fields to be excluded" list loads its schema once instead of re-fetching every model and block on each settings re-render. Internal: extensive additions to the browser-driven E2E suite, plus fixes to the fault-injection lane (DeepL traffic runs through the DatoCMS CORS proxy, which the fault matcher now targets) so the rate-limit/auth-pause/backoff/partial-save behaviors are actually exercised against a real project.
- 3.7.0: Added a translation quality-control layer on top of the 3.6.0 bulk-warnings work. Each AI translation is now verified for completeness — wrong-length, over-split, and truncated responses, lost placeholders, HTML/Markdown structural drift, and likely non-translations are detected, repaired where possible, and surfaced for review (a summary in the sidebar and field dropdown; in bulk, a failed status for content-corrupting issues or a "completed with warnings" status for softer suspicions, plus a persistent post-run review list) instead of being saved silently. Chat-model responses that come back as a single-quoted, trailing-comma, or fence/prose-wrapped array are now recovered instead of failing — fixing records (often ones containing apostrophes) that previously needed several retries to translate. Translations that would overflow a field's character-limit validator are now flagged before save in both the sidebar and bulk flows, instead of silently failing on save; and when DatoCMS does reject a save, the error now names the offending field and reason (e.g. "exceeds its allowed length") rather than a generic message. The bulk run ends with an exportable review report (copy, CSV, or JSON download) listing every record/field/locale that failed or needs review and why — each row now names the record (linked to its editor) and states the real reason, including which link fields were carried over as shared references. Also fixed in this release: JSON fields are now translated structurally (only string values are sent for translation; keys, numbers, and syntax are untouched), fixing saves that failed with a locale-level validation error whenever a model translated JSON keys or broke the syntax; Markdown fields a chat model over-splits are rejoined instead of being cropped to their first block; the DeepL Formality and tag-handling settings in plugin configuration now actually apply to translations; models excluded from the plugin no longer appear in the Bulk Translations page; the sidebar no longer discards a deliberately narrowed target-locale selection when the same source locale is re-picked; and the quality checks raise fewer false alarms on Chinese/Japanese/Korean targets and on prose that starts with a year. Internal: added a browser-driven end-to-end test suite (Playwright + DatoCMS seed fixtures) covering the sidebar, bulk, field-action, records-list, and configuration flows across the provider matrix, plus a Claude-driven self-heal workflow for the suite.
- 3.6.0: Fixed bulk translation of models with a "minimum number of linked records" validation. Previously the whole record failed to translate; and even without the constraint, localized Link/Links fields were left empty in the new locale. Those fields — plus asset galleries with a minimum item count — now carry their source references into new locales instead of being nulled out. Linked records are copied as shared references (not translated, and not followed recursively). The bulk progress modal now flags each affected record with an amber warning icon and a "— with warnings" label, links every record's title (in a new tab) to its editor, and shows the consolidated warning in a hover tooltip — instead of concatenating a wall of text into the status line. An Export CSV button downloads a per-record report (status, CMA timestamp, record ID, title, edit URL, source/target locales, translated field ids, copied link-field ids, and notes). Internal: also fixed stale Anthropic provider tests left over from the 3.5.7 max-tokens key change.
- 3.5.7: Fix for Anthropic CORS and max-tokens issues (thanks @Techwolf12, #148).
- 3.5.6: Fixed AI translations cropping multi-block HTML fields (WYSIWYG / rich-text) to their first block. When a chat provider (OpenAI, Google/Gemini, Anthropic) split a single multi-paragraph field into one array element per block, the surplus blocks were silently dropped on save. They are now rejoined losslessly, and the translation prompt instructs the model to keep each field as a single element. DeepL was unaffected.
- 3.5.5: WYSIWYG / rich-text (HTML) fields are now translated with HTML-aware handling, so markup is preserved instead of being sent and translated as plain text.
- 3.5.4: Fixed Structured Text translation — content blocks nested inside Structured Text fields now translate correctly.
- 3.5.3: Internal: structured request/response debug logging across all providers (OpenAI, Anthropic, Gemini, DeepL), making translation issues easier to diagnose when debugging is enabled.
- 3.5.2: Internal: simplified the translation sidebar component and reworked the debug logger; no change to translation behavior.
- 3.5.1: Fixed translation cancellation and completion reliability. Cancelling a bulk or multi-record run now actually stops it instead of continuing in the background; the progress dialog's Close button no longer stays stuck on "Please wait…" when fewer records come back than were selected; and the single-record sidebar surfaces a clean "cancelled" state instead of an error.
- 3.5.0: Bulk and multi-record translation UX overhaul (select individual models and fields to translate). Also improved Dark Mode support.
- 3.4.6: Fix for SEO "Translate to all locales" functionality setting wrong result-language pairs.
- 3.4.2: Fix for optional & empty localized fields not getting correctly copied to new locales. Empty optional fields will now be null in the new locale, while required fields that cannot be translated will be a direct copy of the source locale's value.
- 3.4.1: Fixed several DeepL bugs from 3.4.0: glossary loading no longer races on duplicate fetches, error detection survives DeepL rewording their messages, and glossary caches now reset when you change API keys. Also fixed a latent bug where empty AI responses crashed translations instead of falling back to the original text.
- 3.4.0: Added better error handling for the CORS proxy and DeepL-specific configuration UI improvements. Selecting a glossary to use should be easier now.
- Prior to 3.4.0: No changelog was kept.
