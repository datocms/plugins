# Repository Guidelines

## Project Structure & Module Organization
- `src/` contains the TypeScript/React plugin code.
  - `src/entrypoints/` hosts the DatoCMS entry screens (`ConfigScreen`, `FieldConfigScreen`, `MediaLayoutsField`).
  - `src/components/` holds UI building blocks (e.g., `AssetCard`, `LayoutBuilder`, `GalleryEditor`).
  - `src/utils/`, `src/hooks/`, and `src/types/` contain shared helpers, hooks, and type definitions.
  - `src/constants.ts` defines aspect-ratio/width presets.
- `public/` contains static assets served by Vite.
- `docs/` stores plugin marketing assets (cover/preview).
- `dist/` is the build output (generated by Vite/TypeScript).

## Build, Test, and Development Commands
- `npm install` installs dependencies.
- `npm run dev` starts the Vite dev server for local UI work.
- `npm run build` runs `tsc -b` and produces `dist/` for production.
- `npm run preview` serves the production build locally.
- `npx tsc -b` performs a standalone type check.

## Coding Style & Naming Conventions
- TypeScript + React with CSS Modules (`*.module.css`).
- Indentation is 2 spaces; strings typically use single quotes; keep trailing commas as in existing files.
- Component folders/files use `PascalCase` (e.g., `src/components/AssetCard/`).
- Hooks are named `useX` and live in `src/hooks/`.
- Prefer co-locating styles with components in `styles.module.css`.

## Testing Guidelines
- There is no automated test runner configured in this package.
- Validate changes with `npm run build` and `npx tsc -b`, then manually test in DatoCMS (field extension UI + config screens).
- If adding tests, keep them close to the feature (`*.test.tsx` alongside components) and document how to run them.

## Commit & Pull Request Guidelines
- Commit messages are short and action-focused; history shows lowercase, simple phrases (e.g., “fix …”, “refactor …”, “version bump”).
- Keep subjects concise (≈72 chars) and avoid noisy scope prefixes unless needed.
- PRs should include: a summary, testing steps, and screenshots/GIFs for UI changes. Note any plugin parameter or data-shape changes and update `README.md`/`docs/` if user-facing behavior changes.
